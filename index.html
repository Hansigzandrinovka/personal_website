Programming Paradigms:
Name Parent Description IOTW:
Imperative None State explicitly what you want from the program AND from where WYSIWIG - what you SAY is what you get
Declarative None State what you want to get, but now how to get it You declare things, but don't define them.
Structured Imperative and Declarative State explicitly (or not), but do NOT use "goto" anywhere The Entire Structure is already laid out for you
Procedural Imperative and Structured Build procedures that can be performed by the entire program Procedural Generation (Minecraft) uses procedures that can behave pseudo-randomly
Functional Declarative and Structured Build functions to be performed by entire program. NO state memory (given input always produces given output) Y = 2x is a function. IF x = 2, Y ALWAYS = 4
Object Oriented Imperative Program uses data structures with methods to call and attributes to hold data. Data structures are instantiated (a box, not the box) We have a box, this box is red, and has a gerbil in it. The other box is blue with a sock in it. They both have six sides.
Logic Declarative Program is an engine that answers questions based on facts and rules It prefers simple logic to complex statements
